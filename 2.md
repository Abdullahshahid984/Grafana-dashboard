For decommissioning services on AKS, the general process we follow is:

1. **Confirm with the App team**  get the list of namespaces, deployments, and services to be removed, and have them confirm nothing is still in use check traffic/metrics first.

2. **Scale down workloads**  scale deployments to 0 replicas as a soft stop before full removal.

3. **Remove Kubernetes resources** delete deployments, services, configmaps, secrets, HPA, ingress/gateway rules, etc. from the namespace.

4. **Remove Istio/ASM configs** if the service has VirtualServices, DestinationRules, or ServiceEntries, those need to be cleaned up too.

5. **Remove Terraform/IaC Workload**  if the namespace or any infra workload identity, federated credentials, role assignments was provisioned via Terraform, remove those from the codebase and run `terraform destroy` or `terraform apply` after removing the resources.

6. **Clean up Azure-side** any managed identities, role assignments, Key Vault access tied to that service.

7. **Update DNS/monitoring**  remove alerts, dashboards, and any DNS records pointing to the service.

We added the RST cluster to conf.yaml which caused the workload onboarding pipeline to create Prometheus alert rule groups for it. However, Azure Backup had already restored those same rule groups into the same resource group with the same name  so when Terraform tried to create them, Azure rejected it with a "resource already exists" error, breaking the DEV pipeline.
As a temporary fix, the conflicting rule groups were manually deleted from the resource group. However the pipeline is still attempting to create Prometheus rule groups for the RST cluster since it is still listed in conf.yaml.
Currently working on a proper Terraform code fix to exclude the RST cluster from Prometheus alert rule group creation. The RST cluster is a temporary validation cluster.

I investigated this thoroughly and the issue is on the application side, not the platform. Here's the evidence:

 **1. Certificate format is valid**
 
 Decoded the actual certificate content from the mounted secret and it's a proper X.509 PEM certificate:
 ```
 -----BEGIN CERTIFICATE-----
 MIIDqDCCApCgAwIBAgIGAZ7wXEH6...
 -----END CERTIFICATE-----
 ```

 **2. Certificate is mounted correctly in the cluster**
 
 - `CATOOLS-OKTA-SAML-CERTIFICATE` is in the `kvcustacctappseus2sit` SecretProviderClass 
 - The synced Kubernetes secret `kvcustacctappseus2sit-secrets-syncd` contains the certificate value 
 - Certificate is available at `/mnt/kvcustacctappseus2sit-secrets/CATOOLS-OKTA-SAML-CERTIFICATE` 

 **3. The error is coming from Spring Boot, not Kubernetes**
 
 ```
 java.lang.IllegalStateException: Missing certificates or unrecognized format
 ```
 This is a Spring Security SAML exception it means the app is not reading the file correctly from the mount path.

 **What you need to check on your side:**
 - Make sure your Spring SAML config is pointing to `/mnt/kvcustacctappseus2sit-secrets/CATOOLS-OKTA-SAML-CERTIFICATE`
 - Make sure it's configured to read it as a file path, not a classpath resource or inline value

 Platform side is all good. The fix is in your application configuration.

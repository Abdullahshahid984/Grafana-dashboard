## Windows (PowerShell)

### Step 1 — Create Certs Folder
```powershell
New-Item -ItemType Directory -Path "C:\certs" -Force
```

### Step 2 — Copy Downloaded Files
```powershell
Copy-Item "$env:USERPROFILE\Downloads\6159-provisioned-tenant-ca.pem" "C:\certs\"
Copy-Item "$env:USERPROFILE\Downloads\netskope chain.p7b" "C:\certs\"
```

### Step 3 — Generate Combined PEM from Windows Store
```powershell
((((gci Cert:\CurrentUser\Root) + (gci Cert:\LocalMachine\Root) + (gci Cert:\CurrentUser\CA) + (gci Cert:\LocalMachine\Root)) | Where-Object { $_.RawData -ne $null } `
| Sort-Object -Property Thumbprint -Unique `
| % { "-----BEGIN CERTIFICATE-----", [System.Convert]::ToBase64String($_.RawData, "InsertLineBreaks"), "-----END CERTIFICATE-----", "" }) `
-replace "`r","") -join "`n" `
| Out-File -Encoding ascii "C:\certs\nscacert_combined.pem" -NoNewline
```

### Step 4 — Set Environment Variables (User Level)
```powershell
[System.Environment]::SetEnvironmentVariable("REQUESTS_CA_BUNDLE", "C:\certs\nscacert_combined.pem", "User")
[System.Environment]::SetEnvironmentVariable("AWS_CA_BUNDLE", "C:\certs\nscacert_combined.pem", "User")
[System.Environment]::SetEnvironmentVariable("NODE_EXTRA_CA_CERTS", "C:\certs\nscacert_combined.pem", "User")
[System.Environment]::SetEnvironmentVariable("SSL_CERT_FILE", "C:\certs\nscacert_combined.pem", "User")
```

### Step 5 — Git
```powershell
git config --global http.sslCAInfo "C:/certs/nscacert_combined.pem"
```

### Step 6 — pip
```powershell
pip config set global.cert "C:\certs\nscacert_combined.pem"
```

### Step 7 — Terraform
```powershell
# Already covered by SSL_CERT_FILE and REQUESTS_CA_BUNDLE env vars set in Step 4
# Verify terraform works
terraform version
```

### Done
```powershell
Write-Host "Done! Restart your terminal." -ForegroundColor Green
```

>  **Restart your terminal** after running all steps for environment variables to take effect.

## WSL (Ubuntu)

> WSL has its own separate trust store from Windows — must be updated independently.

### Step 1 — Create Certs Folder
```bash
mkdir -p ~/certs
```

### Step 2 — Copy Files from Windows into WSL
```bash
# Replace <username> with your Windows username
cp /mnt/c/Users/<username>/Downloads/6159-provisioned-tenant-ca.pem ~/certs/
cp /mnt/c/certs/nscacert_combined.pem ~/certs/

# Verify
ls ~/certs/
```

### Step 3 — Add to Ubuntu System Trust Store
```bash
sudo cp ~/certs/6159-provisioned-tenant-ca.pem \
    /usr/local/share/ca-certificates/netskope-ca.crt

sudo update-ca-certificates --fresh
```

> This covers **curl, wget, netcat, Go binaries** — anything using the Ubuntu system store.

### Step 4 — Set Environment Variables
```bash
echo '' >> ~/.bashrc
echo '# Netskope Certificate - Added June 2026' >> ~/.bashrc
echo 'export REQUESTS_CA_BUNDLE="$HOME/certs/nscacert_combined.pem"' >> ~/.bashrc
echo 'export AWS_CA_BUNDLE="$HOME/certs/nscacert_combined.pem"' >> ~/.bashrc
echo 'export NODE_EXTRA_CA_CERTS="$HOME/certs/nscacert_combined.pem"' >> ~/.bashrc
echo 'export SSL_CERT_FILE="$HOME/certs/nscacert_combined.pem"' >> ~/.bashrc

# Apply immediately
source ~/.bashrc
```
### Step 5 — Git
```bash
git config --global http.sslCAInfo "$HOME/certs/nscacert_combined.pem"
```

### Step 6 — pip
```bash
pip config set global.cert "$HOME/certs/nscacert_combined.pem"
```

### Step 7 — Terraform
```bash
# Already covered by SSL_CERT_FILE and REQUESTS_CA_BUNDLE set in Step 4
# Verify terraform works
terraform version
```

### Done
```bash
echo "Done! Restart your WSL terminal."
```

> **Restart your WSL terminal** after running all steps.

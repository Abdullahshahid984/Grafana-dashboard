## Summary

Add dynamic Workload Identity federated credential support for restore clusters as part of the AKS backup and restore validation initiative.

## Premise

During AKS backup restore validation on `aks-bfhaks-ihub-eus2-dev-rst-01`, all workloads using Workload Identity were stuck in `Init` state after restore. Investigation revealed the following error in pod events:

```
AADSTS700211: No matching federated identity record found for presented assertion issuer
'https://eastus2.oic.prod-aks.azure.com/7a24eae8-33b9-449a-83f5-361634c821ce/76909aed-b958-464f-8a83-02455e1cce47/'
```

**Root Cause:** Every AKS cluster has a unique OIDC issuer URL. The existing workload onboarding pipeline registers federated credentials only for the original cluster at onboarding time. When workloads are restored to a new cluster via Azure Backup, the Managed Identity federated credentials do not cover the restore cluster OIDC issuer. Entra ID rejects the token exchange, causing the CSI Secret Store driver to fail mounting Key Vault secrets.

This issue affects every workload that uses Workload Identity on the restore cluster  not just one namespace.

---

## Changes

### `phases/02-per-instance-per-cluster/variables.tf`
- Added `restore_clusters` variable  accepts a list of clusters (name, resource_group_name, purpose)
- Defaults to `[]`  zero impact when not in use

### `phases/02-per-instance-per-cluster/main.tf`
- Added `data "azurerm_kubernetes_cluster" "restore_clusters"`  dynamically reads OIDC issuer URLs for each restore cluster
- Passed `restore_cluster_oidc_issuer_urls` into `module "kubernetes_resources"`

### `modules/kubernetes-resources/variables.tf`
- Added `restore_cluster_oidc_issuer_urls` variable  map of cluster name => OIDC issuer URL
- Defaults to `{}` — zero impact when not in use

### `modules/kubernetes-resources/main.tf`
- Added `azurerm_federated_identity_credential.workload_identity_restore` resource
- Dynamically creates one federated credential per workload component per restore cluster
- Only created when `restore_cluster_oidc_issuer_urls` is non-empty
- Does not affect the existing `workload_identity` federated credential for the DEV cluster

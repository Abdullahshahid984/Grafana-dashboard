## Summary
Fix ASM revision for the DEV restore cluster (`aks-bfhaks-ihub-eus2-dev-rst-01`) to use a supported service mesh revision.

## Problem
Terraform apply was failing with a `400 Bad Request` error:
> *Revision asm-1-24 is not supported by the service mesh add-on.*

The `service_mesh_profile` was configured with `asm-1-24` which is no longer supported by the Azure Service Mesh add-on.

## Change
Updated the ASM revision from `asm-1-24` to `asm-1-28` to match the existing DEV cluster revision.

## Impact
- Only affects `aks-bfhaks-ihub-eus2-dev-rst-01` (restore cluster)
- No impact on existing DEV, SIT, UAT, or PRD clusters

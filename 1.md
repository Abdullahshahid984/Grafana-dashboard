 Hey found the exact root cause from the logs. Spring is using `PemCertificateParser` to read the certificate from the mount path, and it's failing because the certificate stored in Key Vault has spaces instead of line breaks between the base64 lines. This is a PEM formatting issue.

 The certificate needs to be re-uploaded to KV `kvcustacctappseus2sit` with proper formatting — each base64 line should be 64 characters followed by a newline, like this:
 ```
 -----BEGIN CERTIFICATE-----
 MIIDqDCCApCgAwIBAgIG...
 A1UEAwKQ2FsaWZvcm5p...
 -----END CERTIFICATE-----
 ```
 Please get the correct PEM formatted certificate from your Okta admin portal and re-upload it to KV. That should fix the crash.

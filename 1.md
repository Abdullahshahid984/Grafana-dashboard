## If No NAT Gateway and No Firewall

Then the egress IPs `20.72.112.1` and `20.62.75.29` must be coming from the **Azure Load Balancer outbound rules** that AKS creates automatically.

Check this:

```powershell
# Find the AKS managed resource group
az aks show -n aks-bfhaks-analytics-eus2-prd-01 -g <resource-group> --query nodeResourceGroup -o tsv
```

Then in Azure Portal search for that resource group — it will contain the Load Balancer with the public IPs used for egress.

As part of our AKS backup and restore validation, I provisioned a dedicated restore cluster (aks-bfhaks-ihub-eus2-dev-rst-01). The scope of this exercise involves restoring 71 resources from the dev cluster onto the restore cluster.

To start incrementally, I decided to restore one namespace first ```api-customerlookup-v1``` rather than restoring everything at once.

**What Happened**

**yesterday (api-customerlookup-v1)**
The restore completed but with a warning related to the HPA ScaledObject conflict with KEDA. After the restore, I ran kubectl get pods and noticed the pods were stuck in Init state. Running kubectl get events revealed the following error:

```
AADSTS700211: No matching federated identity record found for presented assertion issuer 'https://eastus2.oic.prod-aks.azure.com/7a24eae8-33b9-449a-83f5-361634c821ce/76909aed-b958-464f-8a83-02455e1cce47/'
```

The root cause is that the restore cluster has a different OIDC issuer URL than the original dev cluster. The Managed Identity (mi-bfhaks-ihub_workloads-api-customerlookup-eus2-dev-01) only has a federated credential registered for the original cluster's issuer  not the restore cluster's. Because of this, Entra ID rejects the Workload Identity token exchange, and the CSI Secret Store driver cannot mount Key Vault secrets, leaving pods stuck in Init.

**Today (api-newaccount-v1 and app-catools-v1)**
Today I restored two more namespaces to further validate:
- ```app-catools-v1```  Restored and running fine
- ```api-newaccount-v1``` Same issue, pods stuck in Init state due to the same ```AADSTS700211``` Workload Identity error

This confirms the issue is not isolated to one workload  any namespace whose Managed Identity is missing a federated credential for the restore cluster's OIDC issuer will hit this blocker.

**Root Cause**

Every AKS cluster has a unique OIDC issuer URL. Federated credentials on Managed Identities are registered per-cluster. When workloads are restored to a new cluster, the existing federated credentials do not cover the new cluster's issuer, causing Entra ID to reject token exchange requests.

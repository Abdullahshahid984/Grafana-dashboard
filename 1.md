**Step 1: Match the ticket endpoint to the actual API in APIM**
Checked the SNOW ticket's endpoint URL (`PUT /v1/accounts/characteristics/compromise-code`), then went into APIM → **APIs** section to find which API definition matched that endpoint confirmed it was **AccountsAPI**.

**Step 2: Check existing environments**
Found that **SIT and UAT already had a product and subscription key** set up for the user. **DEV did not** no product existed there yet.

**Step 3: Create a new product in DEV**
Since DEV was missing, went to the DEV APIM instance (`apim-ads-eus2-entbusops-dev-001`) → **Products** → created a new product (e.g. "Ratndeep Bolegave Subscriber Product") for the user.

**Step 4: Add the required APIs to the DEV product**
Added AccountsAPI (and the other requested APIs CustomerLookup, CustomerCare, Statements, Letters) to this new DEV product.

**Step 5: Create a subscription**
Inside the new DEV product → **Subscriptions** → created a new subscription named after the user, generating a fresh primary key for DEV.

**Step 6: Send all three environment keys**
Compiled the primary keys for **all three environments (DEV, SIT, UAT)** and sent them together to the user via email, marked **Confidential**.

Investigated the volume_snapshot_enabled = true setting and whether it is relevant to our AKS usage.

The setting only snapshots Persistent Volumes backed by Azure Disk (disk.csi.azure.com provisioner). It does not capture Azure Files, Blob-backed volumes, emptyDir, or ConfigMap/Secret mounts.

To determine if we have any qualifying volumes, I ran kubectl get pvc -A and kubectl get storageclass against the cluster. The output shows one active PVC:

  Namespace:     kube-system
  Workload:      cost-analysis-agent-victoria-metrics
  Size:          8Gi
  Access Mode:   RWO
  Storage Class: default (provisioner: disk.csi.azure.com)

The default storage class uses disk.csi.azure.com, which means this PVC is Azure Disk-backed and will be snapshotted on every backup run.

Conclusion: the feature is active and relevant. volume_snapshot_enabled = true is justified. The victoria-metrics volume stores time-series metrics data on disk and will be protected by the snapshot. No change required.

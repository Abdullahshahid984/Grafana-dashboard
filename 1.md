## Summary

Added `exclude_prometheus_alerts` flag support to the instance configuration schema and enabled it for the RST cluster in the DEV instance conf.

## Premise

As part of the AKS backup restore validation fix, the workload onboarding pipeline (`AksPlatform`) was updated to skip Prometheus alert rule creation for clusters marked with `exclude_prometheus_alerts: true`. This PR adds the schema definition for that new field and sets it on the RST cluster in the DEV instance conf.


## Changes

### `schema/instance-conf.yaml`
- Added `exclude_prometheus_alerts` boolean field to `aks_cluster` items properties
- Optional field  no breaking change to any existing `conf.yaml` files that don't use it

### `bfhaksplatform_instances/ihub-dev-eus2/conf.yaml`
- Added `exclude_prometheus_alerts: true` to the RST cluster entry (`aks-bfhaks-ihub-eus2-dev-rst-01`)

Add support for custom Istio Gateway hosts via extra_gateway_hosts

This change introduces support for adding custom (non platform-generated) hostnames to the shared Istio ingress Gateway.

Currently, Gateway hosts are auto-generated based on the workload name and platform ingress subdomain (e.g., <namespace>.<ingress_subdomain>). There is no mechanism to include additional vanity or external domains such as catools-dev.breadfinancial.com.

This PR extends the schema and workload configuration to allow defining additional Gateway hosts per platform instance.

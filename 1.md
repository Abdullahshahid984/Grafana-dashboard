Hi, I have raised a PR for the Workload Identity fix we identified during AKS backup restore validation. During restore validation on the restore cluster ```aks-bfhaks-ihub-eus2-dev-rst-01```, all workloads using Workload Identity were stuck in Init state. The root cause was that each AKS cluster has a unique OIDC issuer URL, and the existing federated credentials on Managed Identities only covered the original DEV cluster not the restore cluster. This caused Entra ID to reject token exchange requests ```AADSTS700211```, blocking Key Vault secret mounts via CSI Secret Store.

Adds a new optional variable restore_clusters to the workload onboarding pipeline. When provided, Terraform automatically creates federated credentials for all workloads on the specified restore cluster. When empty ```default```, there is zero impact on existing behavior. This is fully dynamic supports any cluster name and multiple clusters simultaneously. Cleanup is as simple as setting restore_clusters = [] and re-applying.

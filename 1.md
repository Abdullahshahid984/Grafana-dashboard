## Fix: AKS Trusted Access Role Binding Naming Convention

### Problem

The Trusted Access Role Binding name `backup-aks-bfhaks-ihub-eus2-poc-01` exceeded Azure's 24-character limit and did not follow organizational naming standards.

### Change

Updated the name format to follow `{type}-{app}-{region}-{env}` convention:

```
Before: backup-aks-bfhaks-ihub-eus2-poc-01  ← 38 chars
After:  tarb-bfhaks-eus2-poc               ← 20 chars
```

### How

Used Terraform `split()` and `element()` functions to extract relevant parts from the cluster name key and construct a compliant name.

### Testing

* `terraform plan` verified successfully with no errors
* Name is within Azure's 24-character limit
* Conforms to the organizational naming standard: `tarb-{app}-{region}-{env}`

Additional Fix - install.sh Certificate Generation

Root Cause

The original code used env(EXTRA_SANS_JSON) | from_json inside a single yq expression. By default yq outputs multiline JSON which causes env() to fail with Error: EOF when reading the variable back.

Fix

Split into two yq calls. First build the base dnsNames, then append extra SANs using shell interpolation with -I=0 to force compact single-line JSON output.

Before:
EXTRA_SANS_JSON=$(echo "$INSTALL_TARGET" | yq -o json '.extra_sans // []')
export EXTRA_SANS_JSON
yq '
  .spec.dnsNames = [strenv(INGRESS_SUBDOMAIN), "*."+strenv(INGRESS_SUBDOMAIN)]
    + (env(EXTRA_SANS_JSON) | from_json)
' < "$ROOTDIR/resources/certificate.yaml.tmpl"

After:
EXTRA_SANS_JSON=$(echo "$INSTALL_TARGET" | yq -o json -I=0 '.extra_sans // []')
BASE=$(yq '
  .spec.commonName = strenv(INGRESS_SUBDOMAIN)
  | .spec.dnsNames = [strenv(INGRESS_SUBDOMAIN), "*."+strenv(INGRESS_SUBDOMAIN)]
' < "$ROOTDIR/resources/certificate.yaml.tmpl")
echo "$BASE" | yq ".spec.dnsNames += $EXTRA_SANS_JSON"

The -I=0 flag forces compact output ["catools-dev.breadfinancial.net"] on a single line which shell interpolation handles correctly.

2. CLOUD ENGINEERING AND INFRASTRUCTURE AS CODE

Designed and delivered a comprehensive suite of Terraform IaC modules for the IHUB Azure Data Protection backup platform. Modules delivered include Storage Accounts, Storage Containers, Backup Vaults with CanNotDelete management locks to prevent accidental deletion, Backup Policies with configurable retention schedules and backup frequency, Backup Instances covering all IHUB namespaces incuding future namespaces with Secrets Store CSI enabled, and Backup Extensions. This represented a complete, production-grade backup automation solution built from scratch and deployed successfully across POC and DEV environments.

Authored and deployed Terraform code to configure OIDC Workload Identity for the restore cluster, provisioning Managed Identity Federated Credentials per namespace. This resolved authentication failures caused by OIDC issuer URL mismatches between the DEV and restore clusters, enabling platform components on the restore cluster to authenticate to Azure services correctly without relying on hardcoded credentials or manual intervention.

Designed and implemented a fully dynamic Terraform solution to resolve Workload Identity federated credential gaps across all 71 application namespaces on the restore cluster. The root cause was that every AKS cluster has a unique OIDC issuer URL, and the existing workload onboarding pipeline only registered federated credentials for the original DEV cluster at onboarding time. When workloads were restored to the restore cluster, Entra ID rejected all token exchange requests with AADSTS700211 because the restore cluster OIDC issuer was not registered on any Managed Identity. With 71 namespaces affected, a manual fix was not viable. The solution involved modifying four files in the workload onboarding Terraform codebase, adding a restore_clusters input variable, adding a dynamic data source lookup for restore cluster OIDC issuer URLs, and creating a new azurerm_federated_identity_credential resource using flatten logic to automatically cover all workloads across all specified restore clusters in a single apply. The solution is fully backward compatible with zero impact on existing environments when restore_clusters is not set. A PR was raised for pipeline-based validation in HCP Terraform workspace ws-bfhaks-ihub_main-dev-eus2-01.

Authored and deployed Terraform code to extend the DEV Istio ingress gateway configuration for the catools internal domain, adding extra Subject Alternative Names and custom gateway hosts to support catools-dev.breadfinancial.net. This required modifying the AksPlatform Terraform module to support extra_gateway_hosts and extra_sans input variables, updating the workload onboarding schema and instance configuration files.

Identified and fixed a critical yq multiline JSON parsing bug in the install.sh pipeline script that was causing the certificate generation step to fail with an empty output error. The root cause was that yq outputs multiline JSON by default, which caused the env() expression to fail during pipeline execution. The fix involved adding the -I=0 flag to force compact single-line JSON output and splitting the logic into two separate yq calls to ensure reliable and consistent parsing across all pipeline runs. The fix was validated locally using exported environment variables and a test script before raising the PR.

Implemented data protection namespace exclusion in backup policy scope, preventing circular dependencies between the backup infrastructure and the workloads it protects, and ensuring critical platform namespaces are not inadvertently captured by backup policies.

Raised and coordinated a GSR with the cloud team to provision Private Endpoints for Storage Accounts, scoped to AKS cluster subnets across environments. This eliminates public internet traversal for backup data traffic, strengthens network isolation, and satisfies enterprise security and compliance requirements.

Refactored Terraform configurations to use azurerm_user_assigned_identity data sources instead of hardcoded principal IDs, improving infrastructure portability, reducing module coupling, and aligning with organisational Terraform standards across the platform codebase.

Identified the root cause, applied the appropriate fix, and validated full pipeline recovery, unblocking application team deployments that had been pending on the POC environment.

Investigated and diagnosed persistent Azure DevOps pipeline network timeout failures where both the platform upgrade and workload onboarding pipelines were unable to reach the private AKS API server.

Investigated and resolved a KEDA ScaledObject restore conflict that was blocking namespace restores on the restore cluster. The KEDA admission webhook was rejecting restore operations because an existing HPA was already managing the same deployment as the ScaledObject being restored. Identified the fix, documented it as a known prerequisite step in the restore process, and incorporated it into the restore runbook to prevent recurrence during future restore testing cycles.

Conducted hands-on restore validation testing on the restore cluster, deploying and validating multiple application namespaces to assess restore fidelity. Confirmed that app-catools-v1 restored and ran successfully end-to-end. Identified that api-newaccount-v1 and api-customerlookup-v1 were stuck in Init state post-restore, traced the failures to Workload Identity OIDC issuer mismatch and CSI Secret Store Key Vault mount failures, and used these findings to drive the dynamic Terraform federated credential solution described in Section 2. This validation work produced concrete, documented findings that directly shaped the platform's approach to restore cluster onboarding going forward, and culminated in the authoring of a comprehensive AKS Backup and Restore runbook that the team can now use to execute restore operations consistently and independently.

Authored a comprehensive AKS Backup and Restore runbook documenting the end-to-end restore process, including prerequisite steps such as HPA cleanup for KEDA ScaledObject conflicts, federated credential requirements for Workload Identity on restore clusters, platform component installation order, and namespace validation steps. This runbook serves as the definitive operational reference for the team to execute and repeat restore testing consistently across environments without dependency on tribal knowledge.

Good news, the Helm config for ProfileAPI already supports this. The Key Vault `kvcustacctappseus2sit` is already mounted as a CSI volume in the ProfileAPI pods, so no platform-level changes are needed from our side.
>
> You just need to update your `values.yaml` in your app repo. Find the existing `secretProviders` block for `kvcustacctappseus2sit` and add the following:
>
> **1. Under `secretObjects.data`** (this exposes it as an env var if needed):
> ```yaml
> - key: CATOOLS_OKTA_SAML_CERTIFICATE
>   objectName: CATOOLS-OKTA-SAML-CERTIFICATE
> ```
>
> **2. Under `parameter`** (this tells the CSI driver to pull it from Key Vault):
> ```yaml
> - |
>   objectName: CATOOLS-OKTA-SAML-CERTIFICATE
>   objectType: certificate
>   objectVersion: ""
> ```
>
> Once you deploy, the certificate file will be available inside the pod at:
> `/mnt/kvcustacctappseus2sit-secrets/CATOOLS-OKTA-SAML-CERTIFICATE`
>
> Your application can then read it directly from that file path.
>
> One thing to confirm — in Key Vault, is `CATOOLS-OKTA-SAML-CERTIFICATE` stored under **Certificates** or **Secrets**? If it's stored as a Secret, change `objectType: certificate` to `objectType: secret` in the parameter block. That's the only thing that could differ.

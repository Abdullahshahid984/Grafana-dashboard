 Istio Upgrade (ASM 1.27 → 1.28) 

Hi Team,

I am planning to upgrade Istio (ASM) from version 1.27 to 1.28 across all environments. Please review the scope, key changes, and breaking changes below and confirm your approval to proceed.

---

UPGRADE SCOPE

IHUB Environments
- POC, DEV, SIT, UAT, PERF (CUS), PRD (EUS2), PRD (CUS) — all moving from 1.27 to 1.28

Analytics Clusters
- Non-PRD, PRD (EUS2), PRD (CUS) — all moving from 1.27 to 1.28

---

BREAKING CHANGES — ACTION REQUIRED

Please review these before the upgrade proceeds.

1. InferencePool API promoted to v1
   - Alpha and release candidate versions have been removed.
   - Migrate any existing InferencePool resources to the v1 API type.
   - The field endpointPickerRef.portNumber has been renamed to endpointPickerRef.port.number. Port 9002 is no longer inferred by default.

2. BackendTLSPolicy alpha support removed
   - Only v1 BackendTLSPolicy is supported going forward.
   - The flag PILOT_ENABLE_ALPHA_GATEWAY_API=true is no longer required and should be removed from your configuration.

3. Metric rotation environment variables removed
   - METRIC_ROTATION_INTERVAL and METRIC_GRACEFUL_DELETION_INTERVAL have been removed.
   - Replace with the pod annotation sidecar.istio.io/statsEvictionInterval using the new stats eviction API.

4. ServiceEntry with resolution: NONE — behavior change
   - These entries are now treated as PASSTHROUGH services.
   - During the upgrade, old ztunnel instances may report a NACK in XDS as they do not yet support this new service type. This is expected and self-resolving once ztunnel is updated.

---

KEY HIGHLIGHTS

Security
- seccompProfile (RuntimeDefault) support added for sidecar and validation containers
- Improved root certificate parsing — malformed certificates are now filtered rather than rejecting the entire bundle
- New caCertCredentialName field for improved mTLS certificate management
- MD5 usage removed to align with FIPS 140-3 compliance standards
- Frontend TLS Validation (GEP-91) support added in Gateway API
- Optional NetworkPolicy deployment for istiod (opt-in via global.networkPolicy.enabled=true)

Traffic Management
- Dual-stack (IPv4/IPv6) support promoted to Beta
- Cookie attributes (SameSite, Secure, HttpOnly) now supported in consistent hash load balancing
- Native nftables support in ambient mode as an alternative to iptables
- Wildcard host support in ServiceEntry for HTTP traffic in ambient mode
- Improved socket event handling for better performance under high connection churn

Gateway API and TLS
- Gateway API upgraded to v1.4
- BackendTLSPolicy now supports port-specific TLS configuration via sectionName
- TLS policies can now be applied to external services defined via ServiceEntry

Telemetry
- Upstream spans for gateway traffic enabled by default
- Dual tracing header support (B3 and W3C propagation) for improved interoperability
- New annotations for controlling stats flush and eviction intervals

Installation
- Migrated from Endpoints to EndpointSlice for Kubernetes 1.33+ compatibility
- Persona-based Helm installations introduced (namespace-scoped and cluster-scoped modes)
- Improved revision and tag handling using Kubernetes services, reducing reliance on mutating webhooks in ambient mode
- Kiali addon updated to v2.17.0
